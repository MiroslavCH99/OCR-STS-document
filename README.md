# OCR-STS-document
Сервис для извлечения ФИО владельца из изображений СТС и сохранения результата в CSV.

## Запуск
```bash
uv run main.py --input_folder ./data --output_csv fio_result.csv
```

Опциональные параметры:
- `--debug_candidates` - подробные логи по кандидатам ФИО
- `--recursive` - рекурсивный поиск изображений во вложенных папках

## Архитектура решения
Текущее решение построено как rule-based OCR-пайплайн с многошаговой валидацией кандидатов ФИО.

Компоненты:
- `main.py` - точка входа и CLI-обертка.
- `service.py` - основная логика: OCR-прогоны, извлечение ФИО, скоринг кандидатов, пакетная обработка папки.
- `config.py` - централизованная конфигурация правил (углы поворота, фильтры, пороги, веса скоринга, коррекции OCR-ошибок).
- `PaddleOCR` - движок распознавания текста.
- CSV writer - формирование итогового файла с результатами по каждому изображению.

Поток обработки:
1. Поиск изображений в папке (`iter_image_files`).
2. Для каждого файла выполняются OCR-прогоны на нескольких углах (`0/90/180/270`) и вариантах входа (`raw`, `preprocessed`).
3. OCR-ответ нормализуется в список строк с координатами (`group_ocr_lines`).
4. Находятся кандидаты якоря блока владельца (`СОБСТВЕННИК/ВЛАДЕЛЕЦ`).
5. Из соседних строк извлекаются токены ФИО с фильтрацией адресных/служебных строк и OCR-шума.
6. Формируются кандидаты ФИО, каждый оценивается по интегральному score (количество частей, близость к якорю, штрафы).
7. Выбирается лучший кандидат, результат сохраняется в CSV.


## Идеи как повысить процент распознавания дальше

Более быстрые улучшения:
- Добавить адаптивную предобработку и выбирать лучший вариант по качеству OCR.
- Сделать расширенный набор OCR-вариантовб, например, crop по зоне ФИО.
- Вести ошибки валидации по типам (ошибка якоря, ошибка токенизации, ложное адресное срабатывание) для точечной настройки правил.
- Добавить пост-валидатор ФИО на уровне структуры (фамилия/имя/отчество) с вероятностной моделью вместо правил.
- Ввести словари частотных русских имен/отчеств как soft-weights.
- Использовать вариации двух OCR для сравнения и объединения вариантов.

Долгосрочные улучшения:
- Обучить классификатор строк (name-line vs non-name-line) на размеченном датасете СТС.
